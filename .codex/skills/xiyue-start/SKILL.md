# 息悦生活开发启动

先确认用户当前的单一任务目标。不要将 `docs/02-open-questions.md` 中的历史待确认事项视为可直接开发的需求。

## 加载上下文

按以下顺序读取文档：

1. `docs/03-mvp-and-roadmap.md`：MVP 范围、角色、服务规则和订单状态机。
2. `docs/04-agent-project-spec.md`：项目最高开发规范、权限、技术栈和禁止项。
3. `docs/progress.md`：当前阶段、进行中工作和阻塞项。
4. `docs/decision-log.md`：已采纳的架构决策。
5. `docs/troubleshooting.md`：已记录的问题和解决方式。

按任务需要再读取 `docs/database.md`、`docs/api.md`、`docs/test-cases.md`、`docs/architecture.md` 或 `docs/deployment.md`。

## 开始实施前

先用以下格式说明并等待用户确认（仅纯查询或回答问题时无需确认）：

```text
本次目标：
涉及范围：
实现步骤：
验证方式：
```

每次只实施一个明确、可验证的功能模块。需求、权限、状态流转或数据规则不清楚时，先提问；不得自行扩展。

## 不可违反的规则

- 采用 Spring Boot 单体架构，不引入微服务或未获批准的中间件与框架。
- 用户、阿姨、管理员共用 Vue 前端工程，以角色路由分发。
- 服务时间使用整点小时块；阿姨抢单时才通过数据库唯一索引和事务锁定档期。
- 订单状态机为：待支付、待抢单、待服务、服务中、待确认、待评价、已完成、已取消、投诉中。
- 待支付和待抢单订单不占用阿姨档期；不做独立地址表和超时自动取消。
- 阿姨自行注册，管理员不能新增阿姨，只能管理资料和状态；存在历史订单时禁止物理删除。
- 抢单使用订单状态条件更新保证唯一归属，档期唯一索引只保证同一阿姨时段不冲突。
- 抢单价格以订单快照为准，不与阿姨个人标价比较。
- 服务完成由阿姨提交、用户确认；只有待支付、待抢单、待服务订单可取消。
- 仅待评价订单可投诉一次；处理完成后不再允许评价。
- JWT 密钥、数据库密码、Redis 密码和管理员初始密码不得进入源码、日志或 Git。
- 金额使用 `BigDecimal`，表使用 `utf8mb4`，关键表包含创建时间和更新时间。
- 不删除、重命名、移动、覆盖或回退无关文件和用户已有改动；需要此类操作时先明确征得用户同意。
