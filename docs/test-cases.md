# 测试用例

> 记录系统功能测试、边界测试和异常场景测试。  
> 每个功能完成后，在这里补充对应的测试用例。

---

## 认证与注册

- [ ] 普通用户正确注册（手机号 + 密码 + 123456 + USER）
- [ ] 阿姨正确注册并自动创建一对一 `aunt` 资料（AUNT）
- [ ] 构造 ADMIN 或未知角色注册 → 失败
- [ ] 重复注册（同一手机号）
- [ ] 密码登录成功
- [ ] 验证码登录成功（123456）
- [ ] 错误验证码登录
- [ ] 密码错误登录
- [ ] 找回密码成功
- [ ] 无 Token 请求受保护接口 → 401
- [ ] 无效或过期 Token → 401
- [ ] JWT 角色与数据库当前角色/状态不一致时拒绝越权操作
- [ ] 普通用户访问管理员接口 → 403
- [ ] 阿姨访问用户特有接口 → 403

## 阿姨管理

- [ ] 管理员不能新增或代注册阿姨
- [ ] 管理员编辑已注册阿姨资料
- [ ] 管理员上下架阿姨
- [ ] 下架阿姨不在用户端列表出现
- [ ] 阿姨设置休息状态
- [ ] 休息状态的阿姨不在可抢单列表出现
- [ ] 无历史订单阿姨可逻辑删除
- [ ] 有历史订单阿姨物理删除 → 失败，订单快照仍可查询

## 订单

- [ ] 正常创建订单
- [ ] 创建订单时不锁定阿姨档期
- [ ] 模拟支付成功
- [ ] 待支付订单重复支付 → 失败
- [ ] 已取消订单支付 → 失败
- [ ] 支付后订单进入待抢单
- [ ] 用户取消待支付订单
- [ ] 用户取消已支付待抢单订单（记录模拟退款状态、流水号、时间）
- [ ] 用户取消待服务订单（模拟退款 + 档期释放）

## 抢单

- [ ] 阿姨抢单成功
- [ ] 两个不同阿姨并发抢同一订单 → 条件更新仅一人成功
- [ ] 同一阿姨抢时间冲突的订单 → 档期唯一索引仅允许一次成功
- [ ] 订单更新成功但档期插入冲突 → 整个事务回滚，订单仍为待抢单
- [ ] 阿姨抢下架状态的自己的订单 → 失败
- [ ] 用户取消已抢单订单（含档期释放）

## 服务履约

- [ ] 阿姨开始服务
- [ ] 阿姨提交服务完成
- [ ] 用户确认服务完成
- [ ] 用户取消服务中订单 → 失败并提示联系管理员
- [ ] 用户取消待确认订单 → 失败并提示联系管理员
- [ ] 取消与抢单并发 → 只能有一个状态条件更新成功
- [ ] 取消与开始服务并发 → 只能有一个状态条件更新成功

## 评价

- [ ] 用户提交评分 + 文字评价
- [ ] 同一订单重复评价 → 失败
- [ ] 评价后订单变为已完成
- [ ] 阿姨服务次数和评分更新

## 投诉

- [ ] 待评价订单提交投诉
- [ ] 非待评价订单提交投诉 → 失败
- [ ] 同一订单重复投诉 → 失败
- [ ] 订单状态变为投诉中
- [ ] 管理员查看投诉
- [ ] 管理员处理投诉 → 订单变为已完成且不更新阿姨评分
- [ ] 管理员重复处理投诉 → 失败

## 敏感信息与部署安全

- [ ] 注册、登录和异常日志不输出密码、验证码或完整 JWT
- [ ] `.env`、本地配置、私钥和数据库备份不会被 Git 跟踪
- [ ] 生产环境 MySQL/Redis 不暴露宿主机公网端口
- [ ] 容器健康检查和资源限制生效
